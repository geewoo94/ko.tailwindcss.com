일반적으로 Tailwind CSS v3.0은 Chrome, Firefox, Edge 및 Safari같은 안정된 최신 버전에 맞게 설계되고 테스트됩니다. 하지만 IE 11을 포함한 어떠한 버전의 IE도 지원하지 않습니다.

Tailwind CSS의 대부분의 기능이 모든 최신 브라우저에서 작동하지만, Tailwind에는 `:focus-visible` 의사 클래스 및 `backdrop-filter` 유틸리티와 같은 일부 브라우저가 아직 지원하지 않는 일부 최신 기능에 대한 API도 포함되어 있습니다.

Tailwind는 low-level의 프레임워크이므로 지원하지 않는 CSS 기능을 사용하지 않는 것처럼 지원되지 않는 유틸리티나 한정자를 사용하지 않으면 이러한 기능을 피해 사용하는 것이 쉽습니다.

[Can I Use](https://caniuse.com/?search=visible) 데이터베이스는 익숙하지 않은 CSS 기능에 대한 지원이 확실하지 않은 경우 사용하면 유용한 사이트 입니다..

## Vendor Prefixes

대부분의 CSS 속성은 브라우저에서 벤더 접두사를 이해해야 합니다. 예를 들어, `background-clip: text`는 대부분의 브라우저에서 작동하려면 `-webkit` 접두사가 필요합니다.

```css
.bg-clip-text {
  background-clip: text;
  -webkit-background-clip: text;
}
```

Tailwind CLI 툴을 사용하는 경우 이와 같은 vendor prefix가 자동으로 추가됩니다.

그렇지 않은 경우 PostCSS 플러그인인 [Autoprefixer](https://github.com/postcss/autoprefixer),를 사용하는 것이 좋습니다. PostCSS 플러그인은 사용자가 지원해야 하는 브라우저를 기반으로 필요한 vendor prefix를 자동으로 추가합니다.

사용하기 위해 npm을 통해 설치:

```shell
npm install autoprefixer
```

그런 다음 PostCSS 구성의 플러그인 목록 맨 끝에 추가:

```js
module.exports = {
  plugins: [
    require('tailwindcss'),
    require('autoprefixer'),
  ]
}
```

지원할 브라우저를 지정하는 방법에 대한 자세한 내용은 front-end tooling에서 대상 브라우저를 정의하는 표준 방법인 [Browerslist](https://github.com/browserslist/browserslist)를 참조하십시오.

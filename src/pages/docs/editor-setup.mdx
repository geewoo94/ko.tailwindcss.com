## Syntax 지원

테일윈드 CSS는 `@tailwind`, `@apply`, `@screen`과 같은 많은 사용자 지정 CSS 규칙을 사용하며, 많은 편집기에서 이러한 규칙이 인식되지 않는 경우 경고 또는 오류가 발생할 수 있습니다.

이에 대한 해결책은 일반 CSS 대신 PostCSS 언어 지원을 위한 editor/IDE 플러그인을 설치하는 것입니다. 예를 들어 VS Code에는 Tailwind CSS와 함께 사용할 수 있는 [PostCSS 언어 지원 플러그인](https://marketplace.visualstudio.com/items?itemName=csstools.postcss)이 있습니다.

editor가 CSS 파일에 예상되는 구문에 대해 매우 엄격한 경우 네이티브 CSS linting/validations 검사를 사용하지 않도록 설정해야 할 수도 있습니다.


## IntelliSense for VS Code

Visual Studio Code용 공식 [Tailwind CSS IntelliSense](https://marketplace.visualstudio.com/items?itemName=bradlc.vscode-tailwindcss) extension은 자동 완성, 구문 강조 표시 및 linting과 같은 고급 기능을 사용자에게 제공하여 Tailwind 개발 경험을 향상시킵니다.

<img src={require('@/img/intellisense.png').default} alt="Tailwind CSS IntelliSense extension for Visual Studio Code" width={1344} height={672} className="rounded-lg" />

- **자동 완성**. 클래스 이름뿐만 아니라 [CSS 함수 및 지시문](/docs/functions-and-directives)에 대한 똑똑한 제안입니다.
- **Linting**. CSS와 마크업의 오류와 잠재적인 버그를 강조 표시합니다.
- **Hover 미리보기**. Tailwind 클래스 이름을 마우스로 가리키면 전체 CSS를 볼 수 있습니다.
- **구문 강조**. 테일윈드 기능이 올바르게 강조 표시되도록 구문 정의를 제공합니다.

[GitHub]((https://github.com/tailwindcss/intellisense))에서 프로젝트를 확인하여 자세히 알아보거나 [Visual Studio Code에 추가]((vscode:extension/bradlc.vscode-tailwindcss))하여 지금 바로 시작해 보세요.

## JetBrains IDEs

WebStorm, PhpStorm 등과 같은 JetBrains IDE는 HTML과 '@apply'를 사용할 때 똑똑한 Tailwind CSS 완성을 지원합니다.

[JetBrains IDE의 Tailwind CSS 지원에 대해 자세히 알아보기 &rarr;](https://www.jetbrains.com/help/webstorm/tailwind-css.html)
